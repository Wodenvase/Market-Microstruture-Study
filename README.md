# Endogenous Liquidity & Market Fragility

Research codebase: continuous-time stochastic limit-order market with endogenous liquidity, leverage and margin dynamics.

Contents
- `rust_simulator/` — Rust-based simulator (fast event-driven engine skeleton).
- `python/` — analysis tools (tail fitting, Hurst, plotting).
- `docs/` — LaTeX model specification and notes.

Quick start

Build and run simulator (requires Rust toolchain):

```bash
cd rust_simulator
cargo run --release -- --steps 100000
```

Run Python analysis (requires Python 3.9+):

```bash
python -m pip install -r python/requirements.txt
python python/analyze.py data/sim.csv
```

Repository idea: simulator produces `data/sim.csv` with time series of price, liquidity, volatility proxy and agent-level stats; Python scripts compute tail exponents, Hurst, autocorrelations, and produce diagnostic plots.

See `docs/model.tex` for the formal model specification and equations.

**Methods**

- Model: Continuous-time limit-order market with state vector
	P_t (mid-price), L_t (depth), σ_t (endogenous volatility), agent inventories and leverage. Price increments are driven by order imbalance dI_t with impact dP_t = dI_t / L_t. Liquidity evolves as dL/dt = α - β σ_t L_t - γ InventoryRisk.
- Simulation engine: High-performance Rust binary `rust_simulator` produces CSV output with columns `t,price,depth,sigma,noise_flow,fund_flow,forced_flow,inventory,equity`. CLI options: `--steps`, `--alpha`, `--beta`, `--gamma`, `--seed`, `--output`.
- Demo parameters used for included results: `--steps 5000 --alpha 0.5 --beta 50 --gamma 1.0 --seed 42`.
- Analysis: Python tools in `python/` compute Hill tail estimates, a rough Hurst estimator, rolling volatility and produce diagnostic plots. See `python/analyze.py` and `python/notebooks/analysis.ipynb`.

**Mathematical Results (summary)**

Below are concise analytical statements and derivations underlying the simulation model. See `docs/model.tex` for a fuller write-up.

- State vector: $X_t=(P_t, L_t, \sigma_t, q_i(t), \ell_i(t))$. Price impact is
	\[dP_t=\frac{1}{L_t}\,dI_t.\]

- Liquidity drift (deterministic skeleton):
	\[\frac{dL_t}{dt}=\alpha-\beta\,\sigma_t\,L_t-\gamma\,\mathrm{InventoryRisk}_t.\]

- Endogenous volatility proxy (instantaneous):
	\[\sigma_t^2=\frac{\mathrm{Var}(dI_t)}{L_t^2}.\]

- Equilibrium liquidity condition. At a steady state (starred),
	\[0=\alpha-\beta\,\sigma^*L^*-\gamma\,\mathrm{IR}^* .\]
	If $\sigma^*=S/L^*$ with $S:=\sqrt{\mathrm{Var}(dI_t)}$, the product $\sigma^*L^*=S$ and the equilibrium relation reduces to
	\[\alpha-\beta S-\gamma\,\mathrm{IR}^*=0.\]
	Interpretation: when order-flow variance is exogenous and independent of $L^*$ the liquidity fixed point decouples from $L^*$ — multiplicative effects must enter through $\mathrm{Var}(dI_t)$ or inventory risk to obtain nontrivial dependence and multiple equilibria.

- Linear stability (informal). Linearizing $L_t$ around $L^*$ gives
	\[\frac{d}{dt}\delta L_t\approx -\beta\big(\sigma^*\,\delta L_t + L^*\,\delta\sigma_t\big)-\gamma\,\delta\mathrm{IR}_t.\]
	If $\sigma_t=S/L_t$ then $\delta\sigma_t=-\frac{S}{L^{*2}}\delta L_t=-\frac{\sigma^*}{L^*}\delta L_t$, so the leading linear terms cancel: the simplified model is neutrally balanced and higher-order or delayed inventory feedback is required to produce instability. In practice, endogenous dependence of $\mathrm{Var}(dI_t)$ on $L_t$ or nonlinear inventory feedback yields effective negative damping and can trigger liquidity collapse.

- Bifurcation condition (qualitative). Let the RHS of the $L$-drift be $F(L;\theta)$ with parameters $\theta=(\alpha,\beta,\gamma,\dots)$. Multiple equilibria occur when $F(L;\theta)$ is non-monotone in $L$ (e.g., becomes cubic-like) so that saddle-node bifurcations appear as parameters vary. Economically, increasing sensitivity $\beta$ or reducing baseline supply $\alpha$ moves the system toward the low-liquidity branch and increases fragility.

- Fat-tail mechanism (sketch). Returns satisfy
	\[r\approx \frac{dI}{L}.\]
	Suppose instantaneous order flow $dI$ is thin-tailed (e.g., Gaussian with scale $\sigma_I$) but $L$ is a positive random variable which occasionally becomes very small. Let the small-L tail behave like $\mathbb{P}(L<\ell)\sim C\,\ell^{\kappa}$ for small $\ell$. Then for large thresholds $x$ the return tail is
	\[\mathbb{P}(|r|>x)\approx \int_0^{\infty} \mathbb{P}(|dI|>x\ell)\,f_L(\ell)\,d\ell \sim x^{-\kappa},\]
	i.e. a power-law with exponent $\alpha_{r}=\kappa$. Intuition: extreme returns are generated when small liquidity amplifies ordinary order flow; the rare probability of small L controls the tail index. This is the core mechanism producing heavy tails without heavy-tailed order flow.

- Implication for calibration and simulation: to reproduce realistic tail exponents and volatility clustering the model must (i) allow $L_t$ to fluctuate endogenously into low-liquidity states via inventory- and volatility-sensitive withdrawal, and (ii) include margin/leverage rules that can generate forced flows and cascades.

For full derivations and rigorous statements (spectral/limit theorems, precise conditions for power-law tails), see the analytical notes in `docs/model.tex` and planned extensions in the `docs/` folder.
